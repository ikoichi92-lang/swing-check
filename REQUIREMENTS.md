# swing-check 要件定義書(MVP版 / Phase 1)

## 概要

ゴルフ練習用のスイング自動撮影PWA。スマホを三脚に固定してアプリを起動・放置すると、打球音を検知して自動で直前数秒のスイング映像を切り出し、即座にスローリプレイを画面に表示する。インドアゴルフの自動リプレイシステムと同等の体験をスマホ1台で完結させる。

- 利用者: 本人のみ(個人ツール)
- 対応端末: iPhone(Safari)を最優先。Android Chromeでも動けばなお良い
- サーバー不要。動画データは端末内のみに保存。外部送信は一切しない
- 公開先: GitHub Pages(リポジトリ: https://github.com/ikoichi92-lang/swing-check )

## Phase 1(今回作る範囲)

### 1. 常時録画とリングバッファ

- カメラ映像を常時取得し、直近N秒(設定値、デフォルト5秒)を常にメモリ上に保持する
- MediaRecorderのtimeslice分割、またはフレームバッファ方式など、iOS Safariで確実に動く方式を選定すること
- 解像度は720p目安。フレームレートは端末が出せる最大(30〜60fps)を要求する
- 縦持ち・横持ち両対応(横持ち推奨の旨をUIに表示してよい)

### 2. 打球音検知(インパクト検知)

- マイク入力の音量レベルを監視し、しきい値を超えるスパイクを検知したらインパクトとみなす
- 検知したら「インパクト前X秒+後Y秒」(設定値、デフォルト前3秒・後2秒)を切り出してクリップ化する
- 検知後のクールダウン(デフォルト5秒程度)を設け、同一ショットでの多重検知を防ぐ
- 音量しきい値は設定画面でスライダー調整可能にする。現在の音量レベルをリアルタイムでメーター表示し、しきい値との関係が目で見えるようにする(現場でのキャリブレーション用)
- 手動トリガーボタンも用意する(音検知が使えない環境の保険)

### 3. 自動スローリプレイ

- クリップ生成後、自動で画面にリプレイを再生する(デフォルト0.5倍速、設定で0.25/0.5/0.75/1.0倍を選択)
- リプレイはループ再生。画面タップで一時停止、スライダーでコマ送り(シークバー)
- リプレイ中も裏で録画・検知は継続し、次のショットを打ったらリプレイが新しいクリップに自動で切り替わる
- リプレイ画面に「保存」「破棄」ボタンを表示

### 4. クリップ保存と一覧

- 「保存」したクリップは端末内(IndexedDB)に保存する。保存しなかったクリップは次のショットで自動的に消える
- 保存一覧画面: サムネイル+日時のリスト。タップで再生、長押しまたはボタンで削除
- クリップを端末の写真アプリ/ファイルへエクスポート(ダウンロード)できること(共有・カメラロール保存用)
- ストレージ使用量の目安を表示

### 5. 設定画面

- 音量しきい値(リアルタイムメーター付き)
- 切り出し秒数(前X秒 / 後Y秒)
- リプレイ再生速度
- 検知クールダウン秒数
- 設定はlocalStorageに保存し、次回起動時に復元

### 6. PWA要件

- ホーム画面追加対応(manifest.json、アイコン、スタンドアロン表示)
- Wake Lock APIで画面消灯を防止(セッション中)
- オフラインでも起動できること(Service Workerでアプリ本体をキャッシュ)

## 技術的な制約・注意事項

- カメラ/マイクはHTTPS必須。ローカル開発時はローカルHTTPSサーバーまたは localhost を使うこと
- **iOS Safariの制約を最優先で検証すること。** getUserMedia、MediaRecorderのcodec対応(iOSはH.264/MP4系)、バックグラウンド制限、AudioContextのユーザー操作後開始要件など、iOS固有の落とし穴が多い。実装方式の選定理由をREADMEに残すこと
- リングバッファはメモリ使用量に注意。720p×5秒程度に収める
- 外部ライブラリは最小限。CDN依存は可(オフラインキャッシュ対象にする)
- 単一ページ構成でよい(index.html + js + css。ビルドツール不要ならなお良い)

## Phase 2(今回は作らない。設計時に拡張余地だけ考慮)

- シャフトプレーン自動検出(アドレスフレームからHough変換で直線検出、OpenCV.js想定)+ 端点ドラッグでの手動補正
- 任意ライン描画(直線・垂直線・水平線・円、色分け、全消去)。カメラ固定前提でセッション中は全クリップに同じ線を重畳表示
- Vゾーン表示(シャフトプレーン+肩口プレーンの2本)
- 2動画同期比較再生

## 完成条件(Phase 1)

1. GitHub Pagesで公開され、iPhoneのSafariからURLを開いてホーム画面に追加できる
2. 三脚固定したiPhoneの前で手を叩くと、自動で直前のクリップが切り出されスローリプレイが流れる
3. リプレイの保存・一覧・削除・エクスポートができる
4. 設定変更が反映され、再起動後も保持される

## 開発の進め方

- MVP-first。まず動くものを最短で作り、実地テスト(打ちっぱなし/インドア)のフィードバックで修正する前提
- git管理し、動く単位でコミット。GitHub Pagesへの公開手順(Settings → Pages → mainブランチ選択)もREADMEに記載すること
